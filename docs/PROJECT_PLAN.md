## 1. 项目概述

### 1.1 一句话定义

**CutLab** 是一款面向动画制作流程的本地桌面工具，用于统一管理镜头卡号（Cut ID），提供批量重命名、缺卡检测、自动归档与生产进度导出能力。

### 1.2 解决的核心问题

动画制作中，一个镜头（Cut / 卡）会贯穿分镜、原画、动画、背景、合成等多个环节。实际生产中常见：

- 文件命名混乱：`1卡原画.png`、`C003_key.jpg`、`第5卡分镜.pdf` 并存
- 卡号缺漏难查：C001–C030 范围内哪些镜头缺失，靠人工对表
- 文件夹结构不统一：不同人员、不同环节各自建目录
- 插卡后重编号成本高：C003 与 C004 之间插入新镜头，关联文件名需全部更新
- 制片进度难汇总：无法快速知道哪些卡有原画、哪些缺中间帧

CutLab 将上述重复劳动自动化，让制片、作监、原画在 **同一套卡号体系** 下协作。

### 1.3 与相关概念的区别

| 概念 | 含义 | CutLab 是否覆盖 |
|------|------|-----------------|
| **镜头卡号（Cut ID）** | 标识一个镜头的唯一编号，如 `EP01_S02_C001` | ✅ 核心 |
| **原画卡（Keyframe）** | 镜头内第几帧对应哪张原画（摄影表） | ❌ 不在 MVP；未来可扩展 |
| **分镜卡** | 分镜阶段的镜头编号 | ✅ 作为输入来源之一 |

CutLab 管的是 **镜头级** 的生产文件组织，不是 **帧级** 的原画时间轴。

## 2. 问题与定位

### 2.1 竞品与替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| 手工 Excel + 文件夹 | 零成本、灵活 | 易错、难批量、无联动 |
| 通用批量重命名工具 | 改名快 | 不懂卡号规则、无缺卡逻辑 |
| 制片 ERP / 大型 PM 系统 | 功能全 | 贵、重、学习成本高 |
| **CutLab** | 垂直场景、轻量、本地、可定制规则 | 需自行维护 |

### 2.2 产品定位

```
轻量 · 本地 · 垂直 · 可配置
```

- **轻量**：安装即用，无需服务器，单人或小团队可用
- **本地**：素材文件不出本机，适合未上网或保密项目
- **垂直**：深度理解动画卡号规则与文件夹习惯
- **可配置**：规则、目录模板、识别模式可按项目保存

### 2.3 设计原则

1. **安全优先**：批量操作必须预览、可撤销、支持 dry-run
2. **规则驱动**：卡号格式、目录结构、文件类型均配置化，不写死
3. **核心与 UI 分离**：解析、扫描、重命名逻辑独立于界面，便于 CLI 与 GUI 共用
4. **渐进交付**：MVP 四项闭环先上线，进阶功能按反馈迭代

## 4. 核心功能分级

### 4.1 MVP — 最小可用版本（v1.0）

> 目标：定规则 → 扫文件 → 改名 → 建目录 → 查缺漏，形成完整闭环。

#### F1. 自定义卡号规则

- 内置模式：
  - **简单模式**：`C001`、`C002` …
  - **标准电视动画模式**：`EP{集}_S{场}_C{镜}`，如 `EP01_S02_C001`
- 可配置项：
  - 前缀（项目名、集数、场号）
  - 分隔符（`_`、`-`、无）
  - 序号位数（3 位 / 4 位补零）
  - 文件类型后缀（`_原画`、`_分镜`、`_动画` 等）
- 规则保存为 **项目模板**，切换项目时一键加载

**验收标准**：用户可定义 `{EP:02}_{SC:02}_{CUT:03}_{TYPE}` 类模板，并预览生成结果。

#### F2. 批量规范化重命名

- 扫描指定文件夹（支持递归）
- 自动识别多种「野生命名」：
  - `1卡原画.png`、`第3卡.png`
  - `C001_key.png`、`c003-分镜.jpg`
  - 已有规范名（识别后跳过或校验）
- 预览改名对照表：`旧名 → 新名`，标记 ✓ / ⚠ 无法识别
- **Dry-run** 模式：只预览不执行
- **执行日志** + **撤销（Undo）**：记录每次操作，支持回滚

**验收标准**：100 个混合命名文件，扫描识别率 ≥ 90%（可配置规则补充），批量改名可撤销。

#### F3. 缺卡检测

- 用户输入卡号范围（如 `C001`–`C030`）或集/场范围
- 扫描文件夹，输出：
  - **确认缺失**：范围内无对应文件的卡号
  - **疑似未识别**：有文件但无法解析卡号
- 导出缺失清单（TXT / CSV）

**验收标准**：给定范围与文件集，正确列出所有缺号；不误报已规范命名的卡。

#### F4. 自动归档目录

- 基于卡号清单，一键生成标准目录结构
- 内置默认模板：

  ```
  EP{集}/
    S{场}/
      C{镜}/
        分镜/
        原画/
        动画/
        背景/
        渲染/
  ```

- 支持用户自定义目录模板（JSON）
- 两种模式：
  - **仅建目录**：创建空文件夹结构
  - **建目录并移动**：按文件类型规则移动文件到对应子目录
- 冲突策略：跳过 / 加后缀 / 询问（MVP 默认跳过并报告）

**验收标准**：输入 10 个卡号，一键生成完整目录树；移动模式正确归类且无静默覆盖。

## 5. 产品边界

### 5.1 第一版明确不做

- 云端协作 / 多人在线编辑
- 原画摄影表 / 帧级 time sheet 编辑
- 直接对接 Retas、Toon Boom、CSP 等软件 API
- 视频剪辑、调色、合成
- 用户账号与权限系统

### 5.2 未来可扩展方向

- 与摄影表工具（帧级）数据互通：共享 `C001` 目录
- 插件系统：自定义命名规则、导出格式
- 命令行模式：接入 CI 或批处理流水线
- 可选云同步：项目配置（非素材）备份

## 6. 技术架构

> 详细设计见 **[ARCHITECTURE.md](./ARCHITECTURE.md)**（DDD 领域驱动设计）

### 6.1 技术栈

| 层级 | 选型 | 理由 |
|------|------|------|
| 领域层 | **C# (.NET 10)** | 类型安全、值对象/record 表达力强 |
| 应用层 | Command / Query Handlers | 用例清晰，与 UI 解耦 |
| 桌面 UI | **Avalonia UI** + MVVM | 跨平台、现代 UI |
| 数据存储 | JSON（Infrastructure 持久化） | 本地、无数据库依赖 |
| Excel 导出 | MiniExcel | 轻量、无 Office 依赖 |
| 视频预览 | FFmpeg（v2，Infrastructure 适配器） | 外部进程隔离 |
| 测试 | xUnit · Domain / Application 层 | 领域规则必须可单测 |

### 6.2 DDD 四层架构

```
┌─────────────────────────────────────┐
│  CutLab.App (Presentation)          │  Avalonia + ViewModel
├─────────────────────────────────────┤
│  CutLab.Application (Application)   │  Commands / Queries / Handlers
├─────────────────────────────────────┤
│  CutLab.Domain (Domain)             │  聚合 · 值对象 · 领域服务 · 仓储接口
├─────────────────────────────────────┤
│  CutLab.Infrastructure              │  仓储实现 · 文件系统 · Excel · FFmpeg
└─────────────────────────────────────┘
```

### 6.3 核心聚合（MVP）

| 聚合 | 职责 |
|------|------|
| `AnimationProject` | 命名规范、归档模板、工作根路径 |
| `CutRegistry` | 卡号序列、缺卡检测、插卡（v1.5） |
| `ScanSession` | 文件夹扫描、识别结果、提议新名 |
| `OperationBatch` | 批量重命名/移动/建目录、撤销 |

### 6.4 领域服务

| 服务 | 职责 |
|------|------|
| `INamingService` | 卡号模板渲染 → 目标文件名 |
| `IRecognitionService` | 野生命名 → `CutNumber` 解析 |
| `IGapAnalysisService` | 范围缺卡 + 疑似未识别 |
| `IArchivePathResolver` | 归档目录路径解析 |

## 7. 数据模型

### 7.1 项目配置 `ProjectConfig`

```json
{
  "id": "uuid",
  "name": "新番A",
  "episode": 1,
  "namingRule": {
    "template": "{EP:02}_{SC:02}_{CUT:03}_{TYPE}",
    "separator": "_",
    "typeSuffixes": {
      "storyboard": "分镜",
      "keyframe": "原画",
      "inbetween": "动画",
      "background": "背景",
      "render": "渲染"
    }
  },
  "archiveTemplate": {
    "pattern": "{EP}/S{SC}/C{CUT}/{TYPE}/",
    "folders": ["分镜", "原画", "动画", "背景", "渲染"]
  },
  "recognitionPatterns": [
    "{N}卡{TYPE}",
    "C{CUT:03}_{TYPE}",
    "第{N}卡"
  ],
  "rootPath": "D:/Project/EP01"
}
```

### 7.2 扫描结果 `ScanResult`

```json
{
  "filePath": "D:/raw/1卡原画.png",
  "parsed": { "episode": 1, "scene": 2, "cut": 1, "type": "原画" },
  "proposedName": "EP01_S02_C001_原画.png",
  "status": "recognized | unrecognized | conflict"
}
```

### 7.3 操作日志 `OperationLog`（用于 Undo）

```json
{
  "id": "uuid",
  "timestamp": "2026-06-24T10:00:00",
  "operation": "rename | move | mkdir",
  "entries": [
    { "from": "old/path.png", "to": "new/path.png" }
  ]
}
```

### 7.4 卡号实体 `CutEntry`

```json
{
  "cutId": "EP01_S02_C001",
  "episode": 1,
  "scene": 2,
  "cut": 1,
  "files": [],
  "status": {
    "hasStoryboard": true,
    "hasKeyframe": true,
    "hasInbetween": false,
    "hasRender": false
  },
  "note": ""
}
```

## 11. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 文件名识别规则覆盖不全 | 大量「无法识别」 | 可扩展正则规则库；用户自定义规则 |
| 批量改名误操作 | 文件丢失或混乱 | dry-run + undo log + 冲突跳过 |
| 不同工作室目录习惯差异大 | 归档功能不适用 | 目录模板 JSON 可配置、可分享 |
| FFmpeg 依赖安装复杂 | v2 预览功能门槛高 | 可选组件；检测 FFmpeg 是否可用 |
| 跨平台路径差异 | Windows / macOS 路径问题 | 使用 `Path` API；Avalonia 跨平台测试 |

## 13. 附录

### 13.1 术语表

| 术语 | 英文 | 说明 |
|------|------|------|
| 卡 / 镜头 | Cut | 动画中的一个镜头单元 |
| 卡号 | Cut ID | 镜头的唯一编号 |
| 分卡号 | Cut numbering | 为镜头分配、规范编号的过程 |
| 分镜 | Storyboard | 镜头草图 / 分镜稿 |
| 原画 | Keyframe | 关键帧动画 |
| 动画 | In-between | 中间帧 |
| 缺卡 | Missing cut | 编号范围内缺失的镜头 |
| 插卡 | Insert cut | 在已有序号间插入新镜头 |
| 摄影表 | Exposure sheet | 帧级原画对照表（CutLab MVP 不涉及） |

### 13.2 内置命名识别规则（初版）

| 模式 | 示例 |
|------|------|
| `{N}卡{TYPE}` | `1卡原画.png` |
| `第{N}卡` | `第3卡.png` |
| `C{CUT}_{TYPE}` | `C001_分镜.jpg` |
| `EP{EP}_S{SC}_C{CUT}` | 已是规范名，校验通过 |
| `{N}卡`（无类型） | `5卡.png` → 类型待用户指定 |
