# CutLab 使用说明

> 版本：v1.0 · 镜头卡号生产管理工具  
> 适用平台：Windows 10/11（x64）

---

## 1. 软件简介

CutLab 面向动画制作流程，帮助团队管理**镜头卡号（Cut ID）**相关的素材文件，主要能力包括：

- 扫描文件夹，自动识别卡号、场号、资产类型
- 批量重命名为规范文件名
- 插卡、自动编号、拖拽调整卡号顺序
- 按模板创建归档目录并移动文件
- 缺卡检测、帧序列绑定检测
- 导出 Excel 台账、缺卡清单、预览视频
- 多项目管理、配置导入导出、操作撤销

**核心概念：**

| 术语 | 含义 |
|------|------|
| **项目** | 一套命名规则、归档规则与识别规则的集合，对应一次制作任务 |
| **扫描会话** | 对某个文件夹执行一次扫描后的结果，镜头清单基于此展示 |
| **卡号 / 镜号** | 如 `C001`、`C003`，标识镜头顺序 |
| **资产类型** | 分镜、原画、动画、背景、渲染等 |
| **归档** | 按 `集/场/镜/类型` 等规则整理到目标目录结构 |

## 2. 安装与启动

### 安装方式

详见 [INSTALL.md](INSTALL.md)。简要步骤：

1. 运行 `CutLab-Setup-1.0.0.exe` 安装，或解压 ZIP 便携版
2. 从开始菜单或桌面启动 **CutLab**
3. （可选）安装 [FFmpeg](https://ffmpeg.org/download.html) 并加入系统 PATH，用于生成预览视频

### 开发者运行

```powershell
cd CutLab
dotnet run --project src/CutLab.App
```

## 4. 菜单栏功能一览

### 4.1 「文件」菜单

| 菜单项 | 功能说明 |
|--------|----------|
| **浏览文件夹...** | 打开系统文件夹选择对话框，将路径填入左侧「源文件夹路径」 |
| **扫描** | 对当前源文件夹执行扫描（需已选项目或会自动创建项目） |
| **导出 Excel** | 将当前扫描结果导出为镜头清单 Excel（`.xlsx`） |
| **导出进度台账** | 导出制作进度台账 Excel，按卡号统计各类型完成情况 |
| **导出缺卡清单...** | 将缺卡检测结果导出为 CSV 或 TXT |

### 4.2 「项目」菜单

| 菜单项 | 功能说明 |
|--------|----------|
| **新建项目** | 打开「新建项目」向导，配置规则并绑定素材文件夹 |
| **刷新项目列表** | 重新加载最近项目列表（工具栏下拉框） |
| **项目设置...** | 打开当前项目的详细设置（命名、归档、识别规则等） |
| **导出项目配置...** | 将项目规则导出为 `.cutlab.json`，便于备份或团队共享 |
| **导入项目配置...** | 从 `.cutlab.json` 导入规则到当前项目 |
| **删除当前项目...** | 删除 CutLab 中的项目记录（**不删除**磁盘上的素材文件） |

### 4.3 「操作」菜单

| 菜单项 | 功能说明 |
|--------|----------|
| **执行重命名** | 将已识别文件批量重命名为规范文件名 |
| **插卡重命名** | 在指定卡号后插入新镜，并重命名相关文件 |
| **自动编号（未识别）** | 为未识别文件从起始卡号起依次分配卡号并命名 |
| **创建归档目录** | 按归档模板为已识别镜头创建目录结构（不移动文件） |
| **移动到归档** | 将文件移动到归档目录对应位置 |
| **缺卡检测** | 在指定镜号范围内检测缺失的卡号 |
| **生成预览视频** | 用已识别分镜/图片合成 MP4 预览（需 FFmpeg） |
| **撤销上次操作** | 撤销最近一次成功的重命名/归档/移动操作 |

> 菜单项与工具栏按钮功能一一对应，下文以工具栏简称为主说明。

## 6. 左侧边栏详解

### 6.1 扫描

| 控件 | 说明 |
|------|------|
| **源文件夹路径** | 要扫描的素材根目录，可手动粘贴或点「浏览...」 |
| **浏览...** | 选择文件夹 |
| **扫描** | 执行扫描（与工具栏「扫描」相同） |
| **含子目录** | 勾选后递归扫描子文件夹；不勾选则只扫当前层 |
| **仅预览（不执行重命名/归档）** | 勾选后，点「重命名」「建归档」「移归档」「插卡」「自动编号」**只预览计划**，不真正改文件 |
| **冲突策略** | 目标文件名已存在时的处理方式（见 [§10.1](#101-冲突策略)） |

**扫描支持的文件扩展名：** `.png`、`.jpg`、`.jpeg`、`.psd`、`.pdf`、`.clip`

扫描完成后，中央「镜头清单」会列出所有文件，状态栏显示识别/未识别数量。

### 6.2 编号 / 插卡

| 控件 | 说明 |
|------|------|
| **起始卡号** | 插卡位置或自动编号的起始镜号（1–999） |
| **类型** | 插卡/自动编号时默认的资产类型（分镜、原画、动画、背景、渲染） |
| **插卡时仅未识别文件** | 勾选后插卡只处理未识别文件；不勾选则也处理该镜号上的已识别文件 |
| **自动编号（未识别文件）** | 按文件名排序，从起始卡号起为未识别文件依次分配卡号 |
| **插卡重命名** | 在「起始卡号」后插入新镜（如插在第 3 镜后 → 新镜为 C004 等），并生成对应文件名 |

### 6.3 缺卡检测

| 控件 | 说明 |
|------|------|
| **起始镜号 / 结束镜号** | 检测范围；填 **0** 表示自动按已识别镜号的最小/最大值 |
| **缺卡检测** | 在范围内找出缺失的卡号，结果显示在状态栏右侧 |
| **导出缺卡清单** | 导出 CSV（`.csv`）或纯文本（`.txt`） |

### 6.4 帧序列绑定

用于检查逐帧素材（如 `C001_001.png`、`C001_002.png`）是否缺帧、重复、跨卡等。

| 控件 | 说明 |
|------|------|
| **摘要文字** | 检测后显示问题统计 |
| **帧序列检测** | 按项目设置中的帧序列规则分析当前扫描结果 |
| **导出帧序列问题** | 导出 CSV 或 TXT 问题清单 |

> 需先在 **项目设置** 中启用「帧序列绑定」并配置文件名模板（须含 `{FRAME}`）。

### 6.5 操作历史

| 控件 | 说明 |
|------|------|
| **还可撤销 N 步** | 显示当前可连续撤销的次数 |
| **历史列表** | 最近操作记录：时间、类型（重命名/归档移动/建目录）、成功项数、是否已撤销 |
| **撤销上一步** | 与工具栏「撤销」相同，每次回退一步 |

可撤销的操作：重命名、插卡重命名、自动编号、卡号重排、归档移动、创建目录。

### 6.6 筛选

| 控件 | 说明 |
|------|------|
| **版本标签** | 输入如 `v1`、`draft`、`s`，只显示文件名或元数据中匹配该标签的文件 |
| **应用筛选** | 重新加载镜头清单；再次点击可切换「仅筛选 / 显示全部」 |

## 7. 中央「镜头清单」

### 7.1 列说明

| 列名 | 含义 |
|------|------|
| **卡号** | 识别出的镜头标识，如 `EP01/S01/C003`；未识别显示「（未识别）」 |
| **来源** | 当前文件名 |
| **目标** | 按命名模板生成的目标文件名 |
| **状态** | 见下表 |
| **说明** | 冲突原因、跳过原因等补充信息 |

### 7.2 状态含义

| 状态 | 含义 |
|------|------|
| **Ready** | 可执行重命名/移动 |
| **AlreadyNamed** | 已是规范命名，无需改动 |
| **Conflict** | 目标名冲突（与冲突策略为「遇冲突报错」时） |
| **Skipped** | 跳过（含冲突策略为「跳过」、或无法生成目标名） |
| **Unrecognized** | 未匹配任何识别规则 |
| **AlreadyInPlace** | 归档时文件已在目标位置 |

### 7.3 拖拽调整卡号顺序

1. 在镜头清单中**按住某一行拖动**（需该行有已识别卡号）
2. 拖到目标位置释放
3. 软件会计算卡号重排方案，预览后**自动执行重命名**（非仅预览模式时）

拖动时表头下方会显示蓝色插入线提示。列宽可通过表头分隔条拖拽调整，宽度会自动保存。

## 10. 重要选项说明

### 10.1 冲突策略

当重命名或归档移动时，目标路径已存在文件：

| 策略 | 行为 |
|------|------|
| **遇冲突报错** | 标记为 Conflict，不执行该项 |
| **跳过冲突项** | 标记为 Skipped，继续处理其他文件 |
| **自动加后缀 _1** | 自动使用 `文件名_1.ext`、`文件名_2.ext` 等可用名 |

选择会保存到 `%LOCALAPPDATA%\CutLab\preferences.json`，下次启动保留。

### 10.2 仅预览模式

勾选「仅预览（不执行重命名/归档）」后：

- **重命名 / 建归档 / 移归档 / 插卡 / 自动编号** 只更新镜头清单预览，**不修改磁盘文件**
- 适合大批量操作前先检查目标名与冲突情况

### 10.3 撤销机制

- 每次成功的批量文件操作会记入「操作历史」
- 「撤销」按**时间倒序**逐步回退，每次撤销最近一步
- 已撤销的记录在历史中标记为「已撤销」，不可重复撤销同一步

## 11. 新建项目向导

**入口：** 工具栏「新建」或菜单「项目 → 新建项目」

| 字段 | 说明 |
|------|------|
| **规则模板** | 内置模板（如电视动画标准）；点「应用模板」填充各字段 |
| **项目名称** | 在 CutLab 中显示的名称 |
| **集数 (EP)** | 默认集号，用于命名与归档 |
| **素材文件夹** | 点窗口底部「浏览文件夹...」选择 |
| **归档根路径** | 归档操作的根目录，默认为素材文件夹的上一级 |
| **命名模板** | 如 `EP{EP:02}_S{SC:02}_C{CUT:03}_{TYPE}` |
| **归档路径模板** | 如 `EP{EP:02}/S{SC:02}/C{CUT:03}/{TYPE}` |
| **归档子目录** | 逗号分隔，如 `分镜,原画,动画,背景,渲染` |
| **类型后缀** | 逗号分隔，顺序对应：分镜、原画、动画、背景、渲染 |
| **识别规则** | 每行一条自定义规则（可选） |

点 **「创建」** 后自动切换到新项目并填入素材路径。

## 12. 项目设置窗口

**入口：** 工具栏「设置」或菜单「项目 → 项目设置...」

在新建项目字段基础上，还可配置：

| 字段 | 说明 |
|------|------|
| **加载模板** | 从内置 JSON 模板覆盖当前表单 |
| **命名分隔符** | 默认识别用 `_`，影响部分内置规则 |
| **识别规则** | 每行一条；支持模板 token 或 `^` 开头的正则 |
| **启用帧序列绑定** | 开启帧号检测功能 |
| **帧序列文件名模板** | 如 `C{CUT:03}_{FRAME:03}`，必须包含 `{FRAME}` |
| **最小帧 / 最大帧** | 每镜期望的帧号范围 |
| **默认版本标签** | 如 `v1`，生成文件名时附加（可选） |

修改后点 **「保存」** 生效；下次扫描使用新规则。

命名与归档占位符详见 [NAMING_RULE_SPEC.md](NAMING_RULE_SPEC.md)。

## 13. 典型工作流程

### 流程 A：新剧集从零整理

1. **新建** 项目 → 选模板 → 浏览素材文件夹 → **创建**
2. 勾选「含子目录」（如需要）→ **扫描**
3. 检查镜头清单：未识别文件用 **自动编号** 或手动改文件名后重新扫描
4. 勾选「仅预览」→ **重命名** 检查目标名 → 取消仅预览 → 再次 **重命名** 执行
5. **建归档** 创建目录 → **移归档** 移动文件
6. **缺卡检测** → 必要时 **导出缺卡清单** 发给团队
7. **Excel** / **台账** 导出给制片

### 流程 B：中途插卡

1. 扫描当前文件夹
2. 左侧设置 **起始卡号**（插在该镜之后）和 **类型**
3. 点 **插卡** → 确认预览 → 执行
4. 需要时用 **撤销** 回退

### 流程 C：调整镜头顺序

1. 扫描后，在镜头清单中 **拖拽** 行调整顺序
2. 释放后自动重排卡号并重命名
3. 查看操作历史，必要时撤销

### 流程 D：生成审片预览

1. 确保文件已识别卡号，且为图片格式（png/jpg 等）
2. 安装 FFmpeg 并加入 PATH
3. 点 **预览视频** → 选择保存路径
4. 每镜默认 2 秒，输出 1920×1080 MP4

### 流程 E：团队共享规则

1. **项目设置** 配好规则 → **导出项目配置** 得到 `.cutlab.json`
2. 同事 **导入项目配置** 到本地项目
3. 或在新项目中加载相同模板

## 15. 内置识别规则（摘要）

扫描时按顺序尝试匹配文件名（不含扩展名）：

1. **项目自定义规则**（项目设置中配置）
2. **内置规则**（使用项目 `typeSuffixes` 与分隔符）：
   - 标准规范名：`EP01_S02_C001_原画`
   - 简单卡号：`C003_分镜`
   - 中文卡号：`1卡原画`
   - 序号卡号：`第3卡`（识别镜号，类型可能未识别）

详细语法见 [NAMING_RULE_SPEC.md](NAMING_RULE_SPEC.md)。

## 16. 数据存储位置

| 内容 | 路径 |
|------|------|
| 项目配置 | `%LOCALAPPDATA%\CutLab\projects\`（JSON 文件） |
| 用户偏好（列宽、冲突策略） | `%LOCALAPPDATA%\CutLab\preferences.json` |
| 内置规则模板 | 程序目录 `templates\` |

> 删除项目、卸载软件**不会**自动删除素材文件夹中的 PNG/PSD 等源文件。

## 17. 常见问题

### Q：扫描后全部「未识别」？

- 检查 **项目设置** 中的命名模板、识别规则是否与现有文件名一致
- 尝试 **加载模板** 或参考 `NAMING_RULE_SPEC.md` 调整规则
- 确认文件扩展名在支持列表内

### Q：重命名按钮灰色？

- 先执行 **扫描**
- 等待当前操作完成（状态栏不再忙碌）

### Q：预览视频提示没有 FFmpeg？

- 安装 FFmpeg，确保命令行可执行 `ffmpeg -version`
- **重启 CutLab** 后再试

### Q：预览视频提示没有可用图片？

- 需要**已识别卡号**的文件
- 仅支持 `.png` / `.jpg` / `.jpeg` / `.webp` / `.bmp` 合成视频
- PSD 需先导出为图片，或扫描后先有分镜类 PNG

### Q：归档后想恢复？

- 使用 **撤销**（移归档、重命名均可撤销最近一步）
- 操作历史可查看已执行记录

### Q：仅预览和正式执行有什么区别？

- 仅预览：只更新界面清单，**不改文件**
- 正式执行：实际重命名/移动，并写入操作历史，可撤销
